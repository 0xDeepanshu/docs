# Telegram Integration



Telegram mini apps use an in-app webview and have tight constraints. Gx402 shows how to wire x402 payment flows inside mini apps using the same middleware pattern: create → sign → confirm.

---

## Prerequisites
- Telegram Bot + Mini App setup (webview)  
- Wallet integration strategy (in-app wallet or WalletConnect via browser fallback)  
- Gx402 middleware backend

---

## Flow summary
1. Telegram mini app (webview) calls `/create-payment` on your middleware.  
2. Middleware returns payload.  
3. If an in-app wallet exists, use it to sign; otherwise trigger WalletConnect flow via opening a browser tab or QR flow.  
4. Send signature to `/confirm-payment`. Backend finalizes via x402.  
5. Use Telegram API to update UI/message with result.

---

## Implementation tips
- Use Telegram’s `WebApp` JS SDK to show modals and pass data between webview and bot context.  
- For signing, prefer WalletConnect if in-app wallet is not available.
